## HOW TO USE

1. **Read in order** (00 → 12) — each builds on the previous
2. **Each file follows the same structure:**
   - **Prerequisites** — What you need before this topic
   - **WHY?** — Problem statement, real-world analogy, when to use
   - **HOW?** — Internal mechanisms, algorithms, math, code patterns
   - **WHAT?** — Key concepts, APIs, decision trees, decision frameworks
   - **Patterns & Anti-Patterns** — Quick reference table
   - **Interview Questions** — Top 15 with structured answers
3. **Memorize the decision trees** in §9 of each file — interviewers test trade-off reasoning
4. **Practice the code patterns** — be ready to implement on whiteboard
