## HOW TO USE

1. **Read in order** (00 → 09) — each builds on the previous
2. **Each file follows the same structure:**
   - **WHY?** — Problem statement, real-world analogy, when to care
   - **HOW?** — Internal mechanisms, principles, decision frameworks
   - **WHAT?** — Key concepts, APIs, code patterns, diagrams
   - **PATTERNS & ANTI-PATTERNS** — Quick reference table
   - **INTERVIEW QUESTIONS** — Top 10-15 with structured answers
3. **Memorize the decision trees** in §9 of each file — interviewers test trade-off reasoning
4. **Practice the code patterns** — be ready to implement on whiteboard
